# Gemi — контекст проекта

## Что это

Gemi — веб-приложение для геймификации финансовой грамотности детей 10-12 лет. Расширение к офлайн-настольной игре Cashflow. Дети зарабатывают монеты, торгуют на бирже, развивают бизнес, переводят друг другу средства.

Основная аудитория использует телефоны — мобильный UX в приоритете.

## Стек

- **Frontend:** Vanilla JS, CSS, HTML — никаких фреймворков
- **Backend:** Firebase (Firestore, Auth, Hosting, Cloud Functions на TypeScript)
- **SDK:** Firebase compat v9.22.2
- **Functions:** Node.js, регион europe-west1 (Binance блокирует US IP)

## Деплой

```bash
firebase deploy              # всё (hosting + functions + firestore rules)
firebase deploy --only hosting  # только фронтенд (быстрее)
git push                     # GitHub: MrAleksby/gemi-2.0, ветка main
```

## Архитектура файлов

| Файл | Что делает |
|------|-----------|
| `main.js` | Авторизация, профиль, рейтинг, переводы CF, score_requests, админ-панель |
| `crypto.js` | Биржа: торговля активами, кошельки (биржевой/основной), SL/TP, вывод/пополнение |
| `business.js` | Бизнес: найм сотрудников, кошелёк бизнеса, налоги, вывод |
| `deposit.js` | Инвестиции: депозиты с compound interest |
| `transfer.js` | Переводы монет и CF между игроками |
| `shop.js` | Магазин предметов |
| `chat.js` | Чат между игроками |
| `admin-actions.js` | Одобрение/отклонение score_requests |

## Firestore коллекции

- `users` — профили игроков (coins, exchangeCoins, businessCoins, cf, points, isAdmin)
- `transactions` — история переводов
- `score_requests` — заявки на начисление статистики (games, wins, cf, points, coins)
- `tax_log` — история налогов
- `deposits` — активные депозиты
- `exchange_trades` — история сделок на бирже
- `businesses` — данные бизнесов игроков

## Валюты и правила округления

- **Монеты (coins)** — основная валюта, округляются до **сотых** (`Math.round(x * 100) / 100`)
- **CF** — игровая валюта, **не округлять**, курс 1 CF = 1000 сум
- **Опыт (points/XP)** — целые числа, **не трогать**
- **KD и рейтинг** — **не трогать**

## Роли

- **Игрок** — обычный пользователь
- **Админ** — определяется по полю `isAdmin === true` в документе users (не по имени)

## Технические правила

- После транзакций Firestore читать данные с `{ source: 'server' }` — иначе SDK отдаёт кеш
- Кнопки "Макс" в формах ввода суммы **убраны** — не добавлять
- Поля ввода суммы: `width:100%; padding:10px 12px; border:1.5px solid #ddd; border-radius:10px; font-size:1em`
- Налог на вывод с биржи: **1%**, минимум 0.01 монеты
- Cloud Functions деплоятся в europe-west1 (не us-central1)
